This project converts a CSV file into a **class-based OWL ontology** using **Python**, **pandas**, and **Owlready2**.

It reads three columns from the CSV:

- `Dept/Roll-up Name`
- `System Name`
- `Symptom/Component Name`

and builds an ontology with:

- `Department`
- `System`
- `Symptom`

as top-level classes.

Each row contributes to subclass relationships and logical restrictions.

`mnr_ontology_rebuild.py` reconstructs the ontology based on the original OWL file generated by Protégé. To make the code easier to read and maintain, ontology classes, properties, subclass relationships, and restrictions are defined separately using Owlready2.

Considering that the code was directly generated by AI and can read the CSV file automatically, this approach is more convenient than building the ontology manually in Protégé.

The OWL format generated by Owlready2 does not differ significantly from the OWL format generated by Protégé.
